# RTC (PL031) — Design

Date: 2026-06-13. Status: approved design, ready for an implementation plan.

## Context

Sub-project **C** of the "full device model" milestone (A = DeviceManager framework,
B = virtio-rng, both merged). This adds an ARM PrimeCell PL031 real-time clock — the
first *plain-MMIO* device since the 16550 serial, and the consumer of the `Pl031`
FDT kind that sub-project A deliberately left for this task.

Purpose: give the guest correct wall-clock time at boot (so `date` is right) and a
correct clock after restore (the monotonic generic timer is re-anchored on restore,
but a clock that reads host wall-time is always current regardless).

### Existing pieces this builds on

- `devices::device`: `FdtKind { Ns16550a, VirtioMmio }` (this adds `Pl031`),
  `DeviceMgrError`, `trait MmioDevice: BusDevice { fdt_kind; snapshot_id; save;
  restore }`. `Serial` is the model for a plain-MMIO `MmioDevice`.
- `devices::bus::BusDevice`: `read(&mut self, base, offset, &mut [u8])`,
  `write(&mut self, base, offset, &[u8])`.
- `arch::aarch64::fdt`: `enum FdtDevice { Serial(MmioDev), VirtioMmio(MmioDev) }`
  (this adds `Rtc(MmioDev)`), `MmioDev { addr, size, irq }`, node generators, and a
  `CLOCK_PHANDLE` for the `apb_pclk` fixed-clock the serial node references.
- `vmm::device_manager::DeviceManager`: `add`/`add_restored`/`fdt_devices` (maps
  `FdtKind` → `FdtDevice`).
- The boot harness wires devices via `mgr.add` (fresh) and an id-keyed
  `add_restored` loop (restore).

## Goal

A time-only PL031 RTC, always-on, that reads host wall-clock seconds (plus a
guest-settable offset), binds the kernel's `rtc-pl031` driver, and snapshots its
offset. Adding it = the `Pl031` FDT kind + node generator + one device file + one
`add` call + one restore arm; no snapshot-format change.

Non-goals: the alarm/Match register and RTC interrupt (stubbed, no IRQ);
sub-second precision (PL031 is whole-seconds by hardware definition); writing host
time back from the guest (Load only shifts the guest's view via an offset).

## Architecture

### `crates/devices/src/rtc.rs` (new) — `Pl031`

```rust
pub struct Pl031 {
    /// Seconds added to host wall-clock time to form the value the guest reads in
    /// RTCDR. 0 means "report host wall-clock". A guest write to RTCLR sets this so
    /// the guest's chosen time is honored: offset = loaded - now_unix().
    offset: i64,
    /// Last value written to RTCLR (returned on RTCLR reads; cosmetic).
    load: u32,
    /// Last value written to RTCMR (returned on RTCMR reads; alarm is not wired).
    match_reg: u32,
    /// Last value written to RTCCR (control); reads report enabled regardless.
    control: u32,
    /// Interrupt mask (RTCIMSC); stored for read-back, never acted on.
    imsc: u32,
}
```

`Pl031::new() -> Self` (all fields 0).

Host time: `fn now_unix() -> i64` =
`SystemTime::now().duration_since(UNIX_EPOCH)` seconds as `i64` (saturating to 0 if
the clock is before the epoch — impossible in practice).

Current RTC value: `fn rtcdr(&self) -> u32 { (now_unix() + self.offset) as u32 }`
(PL031 is a 32-bit seconds counter; wrap in 2106 is by hardware design).

#### Register map (offsets from the device base; all 32-bit, little-endian)

| Off | Name | Read | Write |
|-----|------|------|-------|
| 0x00 | RTCDR | `rtcdr()` | ignored |
| 0x04 | RTCMR | `match_reg` | store `match_reg` (no alarm effect) |
| 0x08 | RTCLR | `load` | `load = v; offset = v as i64 - now_unix()` |
| 0x0C | RTCCR | `1` (RTCEN/enabled) | store `control` (no effect) |
| 0x10 | RTCIMSC | `imsc` | store `imsc` |
| 0x14 | RTCRIS | `0` | — |
| 0x18 | RTCMIS | `0` | — |
| 0x1C | RTCICR | `0` | ignored (interrupt clear; no IRQ) |
| 0xFE0 | PeriphID0 | `0x31` | — |
| 0xFE4 | PeriphID1 | `0x10` | — |
| 0xFE8 | PeriphID2 | `0x04` | — |
| 0xFEC | PeriphID3 | `0x00` | — |
| 0xFF0 | PCellID0 | `0x0D` | — |
| 0xFF4 | PCellID1 | `0xF0` | — |
| 0xFF8 | PCellID2 | `0x05` | — |
| 0xFFC | PCellID3 | `0xB1` | — |

The PeriphID/PCellID bytes are the standard PL031 PrimeCell identification (same
values QEMU uses); the kernel's amba bus reads them to bind `rtc-pl031`.

#### `BusDevice` impl

- `read`: only 4-byte reads are meaningful (Linux accesses these registers as u32).
  Compute the register value per the table, write it little-endian into `data` for
  `data.len() == 4`. For any other width or an unmapped offset, zero-fill `data`
  (defensive; the driver never does this). ID registers return their byte
  zero-extended to u32.
- `write`: for `data.len() == 4`, decode the u32 and apply per the table. Other
  widths / unmapped offsets are ignored (logged at debug).

#### `MmioDevice` impl

- `fdt_kind() -> FdtKind::Pl031`
- `snapshot_id() -> "rtc"`
- `save() -> serde_json::to_value(Pl031Snapshot { offset })` — only `offset` affects
  the time the guest reads; `load`/`match_reg`/`control`/`imsc` are cosmetic
  read-back and are intentionally not snapshotted.
- `restore(v) -> { self.offset = Pl031Snapshot::deserialize(v)?.offset; Ok(()) }`,
  mapping a serde error to `DeviceMgrError::StateInvalid { id: "rtc", reason }`.

`Pl031Snapshot { offset: i64 }` — a small serde struct.

### `devices::device` — `FdtKind::Pl031`

Add the `Pl031` variant to `FdtKind` (deferred from sub-project A). It already
derives serde + Copy + Eq, so the snapshot `DeviceRecord` carries it unchanged.

### `arch::aarch64::fdt` — `FdtDevice::Rtc` + `create_rtc_node`

- Add `Rtc(MmioDev)` to `FdtDevice`; `generate`'s match gains
  `FdtDevice::Rtc(m) => create_rtc_node(&mut fdt, m)?`.
- `create_rtc_node` (mirrors `create_serial_node` minus interrupts):
  ```
  node "rtc@{addr:x}"
  compatible = ["arm,pl031", "arm,primecell"]   // string list
  reg = [addr, size]
  clocks = CLOCK_PHANDLE
  clock-names = "apb_pclk"
  // NO "interrupts": time-only, alarm not wired.
  ```
  The `arm,primecell` compatible drives the amba probe that reads the PrimeCell IDs.

### `vmm::device_manager` — kind mapping

`fdt_devices` match gains `FdtKind::Pl031 => FdtDevice::Rtc(m)`.

## Wiring (`spike/src/bin/boot.rs`)

Always-on, added after serial on both paths (window/SPI/FDT-node all from the
framework; the allocated SPI is unused since the node lists no interrupt):

- Fresh boot: `mgr.add(layout::MMIO_WINDOW, |_irq| Pl031::new()).expect("add rtc");`
  (the build closure ignores `irq` — RTC asserts no interrupt).
- Restore: a `"rtc"` arm → `mgr.add_restored(rec, |_irq| Pl031::new())`.

## Snapshot

`offset` is the only state; `MmioDevice::save/restore` on `Pl031` handles it. No
snapshot-version bump — the device-record list is self-describing, so a pre-RTC
snapshot restores without an RTC and a new one carries the `"rtc"` record. On
restore the guest reads host-now + offset = correct current wall time.

## Error handling

- Non-4-byte or unmapped-offset reads → zero-fill (never happens from the driver).
- Unmapped/wrong-width writes → ignored.
- `restore` with malformed state → `DeviceMgrError::StateInvalid`.

## Testing

Unit (in `rtc.rs`, no entitlement):
1. **DR ≈ now.** `Pl031::new()`, read 0x00 as u32 → within ±5 s of the test's own
   `SystemTime::now()` unix seconds.
2. **Load sets the clock.** Write 0x08 = `T` (a fixed far-future value, e.g.
   2_000_000_000), then read 0x00 → within ±5 s of `T` (offset applied).
3. **PrimeCell IDs.** Reads at 0xFE0/0xFE4/0xFE8/0xFEC and 0xFF0/0xFF4/0xFF8/0xFFC
   return `0x31,0x10,0x04,0x00` and `0x0D,0xF0,0x05,0xB1` respectively (low byte of
   the u32).
4. **CR enabled.** Read 0x0C → 1.
5. **Snapshot round-trip.** Write Load=`T`, `save()`, build a fresh `Pl031`,
   `restore(saved)`, read 0x00 → within ±5 s of `T` (offset survived).
6. **Identity/FDT.** `fdt_kind() == Pl031`, `snapshot_id() == "rtc"`; in `fdt.rs`,
   `create_rtc_node` via `FdtDevice::Rtc` produces a node whose `compatible` first
   string is `arm,pl031` and that has no `interrupts` property.

Live (drivers + interactive): `date` in the guest shows the correct year;
`dmesg | grep rtc-pl031` shows it bound (`rtc-pl031 ...: registered as rtc0`);
`scripts/restore_test.py` / `restore_clone_test.py` still pass.
